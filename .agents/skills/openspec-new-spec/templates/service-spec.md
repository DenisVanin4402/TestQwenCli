<!-- ПРИМЕР:
### Правила расчёта итоговой стоимости

1. **Расчёт `subtotal` для каждой позиции:**
   `subtotal[i] = items[i].price * items[i].quantity`

2. **Расчёт `originalAmount`:**
   `originalAmount = SUM(subtotal[i] for all i)`

3. **Расчёт `finalAmount` (после применения скидки):**
   ```
   IF promoCode применён И валиден:
       discountAmount = вычислить по правилу 2.3.4
       finalAmount = MAX(0, originalAmount - discountAmount)
   ELSE:
       discountAmount = 0
       finalAmount = originalAmount
   ```
   При округлении использовать BANKERS_ROUNDING до 2 знаков после запятой.

### Правила применения промокода

2. **ЕСЛИ** поле `promoCode` не передано или пустое,
   **ТОГДА** заказ оформляется по полным ценам, `discountAmount = 0`.

3. **ЕСЛИ** `promoCode` передан,
   **ТОГДА** проверить промокод в таблице `promo_codes`:
   - **ЕСЛИ** промокод не найден, **ТОГДА** вернуть ошибку 400 "Промокод не найден".
   - **ЕСЛИ** `valid_until < now()`, **ТОГДА** вернуть ошибку 400 "Срок действия промокода истёк".
   - **ЕСЛИ** `usage_count >= max_usages`, **ТОГДА** вернуть ошибку 400 "Промокод больше не действует".

4. **Расчёт скидки:**
   - **ЕСЛИ** `discount_type = 'PERCENT'`,
     **ТОГДА** `discountAmount = ROUND(originalAmount * discount_value / 100, 2)`.
   - **ЕСЛИ** `discount_type = 'FIXED'`,
     **ТОГДА** `discountAmount = MIN(discount_value, originalAmount)`.
   - В обоих случаях гарантируется инвариант 1.3.2 (`discountAmount <= originalAmount`).

### Правила резервирования товара

5. **ЕСЛИ** при создании заказа хотя бы один товар имеет `available_quantity < requested_quantity`,
   **ТОГДА** заказ НЕ создаётся, возвращается 409 CONFLICT со списком недоступных позиций.
   **ИНАЧЕ** для каждой позиции выполняется `available_quantity -= quantity` атомарно.

### Правила отправки уведомлений

6. **ЕСЛИ** заказ перешёл в статус `PAID` (см. переход 2.4.3),
   **ТОГДА** отправить событие `ORDERS.PAID` в Kafka — notification-service отправит email "Заказ оплачен".
7. **ЕСЛИ** заказ перешёл в статус `SHIPPED`,
   **ТОГДА** отправить событие `ORDERS.SHIPPED`.
8. **ЕСЛИ** заказ перешёл в статус `CANCELLED`,
   **ТОГДА** отправить событие `ORDERS.CANCELLED`.
