# /sdd:tasks

Разбей технический план на граф проверяемых задач.

## Входные данные

Путь к спецификации или `{{args}}`

## Что читать

1. `docs/specs/000N-slug/plan.md` — технический план
2. `docs/specs/000N-slug/spec.md` — требования (REQ-IDs)
3. `docs/specs/_template/tasks.md` — шаблон задач

## Что делать

1. Создать граф задач, где каждая задача ссылается на REQ-ID
2. Каждая задача должна иметь:
   - Уникальный ID (TSK-1, TSK-2, ...)
   - Чёткий вход и выход
   - Зависимости от других задач
   - Оценку сложности (S/M/L)
3. Выявить задачи, которые можно выполнять параллельно
4. Определить ownership и блокировки
5. Создать или обновить `docs/specs/000N-slug/tasks.md`

## Разрешено изменять

- `docs/specs/000N-slug/tasks.md`

## ЗАПРЕЩЕНО изменять

- Код
- Spec или plan

## Gate

Остановиться после создания tasks.md. Предложить следующий шаг:
- `/sdd:implement` для начала реализации
