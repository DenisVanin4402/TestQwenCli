# SDD Task Slice

## Входные данные

- `docs/specs/<id>-<slug>/spec.md`
- `docs/specs/<id>-<slug>/plan.md`

## Процедура

1. Прочитать spec и plan.
2. Выявить независимые компоненты/модули изменений.
3. Для каждого компонента создать задачу с:
   - REQ-ID ссылкой
   - Чётким входом и выходом
   - Зависимостями от других задач
   - Оценкой сложности
4. Построить граф зависимостей.
5. Выявить задачи для параллельного выполнения.
6. Создать `tasks.md`.

## Результат

- `docs/specs/<id>-<slug>/tasks.md` с графом зависимостей
- Каждая задача имеет вход/выход/REQ-ссылку

## Принципы нарезки

- Одна задача = одно изменение, которое можно верифицировать.
- Не резать слишком мелко (каждый TSK — > 5 минут работы).
- Не резать слишком крупно (каждый TSK — должен иметь один понятный выход).
