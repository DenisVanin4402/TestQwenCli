# SDD Test Gap Analysis

## Входные данные

- `docs/specs/<id>-<slug>/spec.md` (REQ и AC)
- `docs/specs/<id>-<slug>/test-plan.md`
- Текущие тестовые файлы

## Процедура

1. Прочитать spec — выписать все AC.
2. Прочитать test-plan — какие AC должны быть покрыты какими тестами.
3. Найти существующие тестовые файлы через grep/glob.
4. Для каждого AC проверить:
   - Есть ли тест?
   - Что проверяет тест?
   - Покрывает ли AC полностью?
   - Является ли тест expected failing (до реализации)?
5. Создать report с пропусками.

## Что проверять

| AC | Тест | Покрывает? | Замечания |
|----|------|------------|-----------|
| AC-1 | test_method() | partial | Не тестирует edge case |
| AC-2 | missing | no | КРИТИЧЕСКИЙ ПРОПУСК |

## Результат

- Список AC без тестового покрытия
- Список AC с частичным покрытием
- Рекомендации по дополнительным тестам
- Файл: `docs/specs/<id>-<slug>/test-plan.md` обновлён

## Ограничения

- Не оценивать качество тестов, только наличие покрытия AC.
- Не писать сами тесты (это задача Builder или Test Engineer).
