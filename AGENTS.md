# Инструкции для AI Coding-агентов

Этот файл содержит кросс-инструментальные инструкции для всех coding-агентов, работающих в этом проекте.

## Source of Truth

- Требования живут в `docs/specs/`.
- Нельзя реализовывать поведение, не описанное в спецификации или явно не запрошенное пользователем.
- Если код, тесты и спецификация расходятся — остановить работу и сообщить о несоответствии.
- Конституция SDD (`docs/sdd/constitution.md`) содержит непереговорные принципы.

## SDD Workflow

1. Изучить существующий код перед планированием изменений.
2. Определить соответствующую спецификацию, тесты, контракты и публичные интерфейсы.
3. Выполнить минимальные скопированные изменения.
4. Запустить команды верификации.
5. Отчитать об изменённых файлах, командах и residual risks.

## Обязательные команды верификации

```bash
# Компиляция
mvn compile

# Все тесты
mvn test

# Упаковка
mvn package

# Проверка SDD-артефактов
bash scripts/sdd-lint.sh
```

## Ограничения

- Не добавлять новые зависимости без ADR или явного разрешения.
- Не менять публичные контракты без обновления `contracts/` и тестов.
- Не редактировать миграции ретроспективно без явного запроса.
- Не писать секреты, токены или пароли в код, логи или артефакты.
- Реализация не начинается, пока критические требования содержат `[NEEDS CLARIFICATION]`.

## Контекстный пакет

Перед выполнением задачи получить context packet из `docs/sdd/context-management.md`:

```
spec -> plan -> tasks -> tests -> implementation -> verification
```

## Отчёт о завершении задачи

При завершении задачи предоставить:

1. Список изменённых файлов
2. Запущенные команды и их результаты
3. Spec compliance notes (какие REQ/AC выполнены)
4. Residual risks (что ещё может потребовать внимания)
5. Обновления `task-state.md` и `work-log.md`
